Nano pi/树莓派及arm架构，Ubuntu下找不到包问题

2020-05-13 21:14:37
  • Nano pi，树莓派，Ubuntu

近日，卑微的小编68米在某鱼淘了一个nanopi 2Fire（烈火版）的开发板，用来做服务器+nas。

板子就长这样，然后sd卡烧写Ubuntu 系统，正常开机，但是 快节奏的我怎么能满足于 官方源的龟速呢。所以，就换源嘛 由于开发板使用的是arm架构 ，而百度的ubuntu国内源 基本都是基于x86的。于是，就出现了下边的问题。

解决办法

首先打开配置源的文件

vim /etc/apt/sources.list

将官方源注释（前方加#）并加入以下源，亲测可用，理论基于arm架构的开发板都可用。

deb http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial main multiverse restricted universe
deb http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-backports main multiverse restricted universe
deb http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-proposed main multiverse restricted universe
deb http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-security main multiverse restricted universe
deb http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-updates main multiverse restricted universe
deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial main multiverse restricted universe
deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-backports main multiverse restricted universe
deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-proposed main multiverse restricted universe
deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-security main multiverse restricted universe
deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ xenial-updates main multiverse restricted universe

然后 保存退出。

sudo apt-get update

输入上边命令，可以看到比官方源快了很多。
